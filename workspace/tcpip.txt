#inclue <sys/socket.h> 시스템 소켓에관한 헤더파일
int socket(int domain, int type, int protocol); 소켓함수 int 타입으로 리턴을받음

성공시 파일 디스크립터, 실패 시 -1 반환 파일 디스크립터는 파일을 프로세스에 할당할때

그 파일을 구분하기위해 놓은 인덱스이다.

domain은 어떤 영역에서 통신할 것인지에 대한 영역을 지정해준다. 예를들어 프로토콜 내부에서

인터넷 프로토콜을 사용할것인지...

type은 어떤 타입의 프로토콜을 사용할 것인지에 대해 설정하는 것 입니다.

전송제어프로토콜(TCP), DP(사용자 데이터그램 프로토콜), 사용자정의

protocol도 IPPROTO_TCP(TCP일때),IPPROTO_UDP(UDP 일때)

int bind(int sockfd, struct sockaddr *myaddr,socklen_t addrlen);
성공시 0, 실패시 -1 반환

socket에서 반환된 socket desciptor를 받고 소켓에 구조체 포인터를 받는다.

그리고 구조체에 길이를 받는다.

bind는 소켓에 주소를 할당해주기 위한 시스템 콜이다.

struct sockaddr에는 나의 adddress와 port number가 들어있다.

socklen_t addrlen는 구조체의 길이를 반환하는대 정확히 어떤 구조체의 길이를 반환하는진 모른다.

struct sockaddr에는 소켓에 길이와 도메인 주소가 들어있다.(내가 서버라는 가정하에 내 주소를 말하는것같다.)


포트넘버 그리고 ip주소가 들어있다.
32bit타입 ip주소도 있다.
